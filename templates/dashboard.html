System Architecture - Design Patterns
Factory Pattern

Used for creating different user types (General, Provider, Admin) based on role selection during registration.

Singleton Pattern

Ensures only one ServiceRegistry instance manages all public services throughout the application.

Strategy Pattern

Provides interchangeable search strategies (Category, Location, Rating) for flexible service discovery.

Facade Pattern

Simplifies complex subsystem interactions (Search, Detail, Map services) through a unified interface.

Observer Pattern

Enables real-time notifications when services are added, updated, or approved, keeping users informed.
